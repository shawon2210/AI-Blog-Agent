# AI Blog Agent — Implementation Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Build an auto-correcting blog writing AI agent using Google ADK that plans, validates, writes, and self-corrects a full blog post from a user topic.

**Architecture:** A multi-agent pipeline with a root orchestrator (`blogger`) that dispatches two robust loop-agents (`robust_blog_planner` and `robust_blog_writer`). Each loop agent wraps an LLM agent + a validation checker in an automatic retry loop (up to 3 attempts). The root agent then enriches the final output with titles and social hooks. All agents communicate via a shared state dictionary (`blog_outline`, `blog_draft`). A FastAPI web server exposes the agent over HTTP, and a minimal React frontend provides a topic input + real-time agent trace view.

**Tech Stack:** Python 3.12, Google ADK (`google-genai` + `google-adk`), FastAPI, Uvicorn, React (Vite), Pydantic, pytest, uv.

```python
"""Validation agents for self-correction loops."""
from src.models import BlogOutline

def validate_outline(outline: BlogOutline | None) -> bool:
    """Check that outline has all required parts."""
    if outline is None:
        return False
    if not outline.title or not outline.title.strip():
        return False
    if not outline.introduction or not outline.introduction.strip():
        return False
    if len(outline.sections) < 4:
        return False
    if not outline.conclusion or not outline.conclusion.strip():
        return False
    return True

def validate_draft(draft) -> bool:
    """Check that draft has sufficient content."""
    if draft is None:
        return False
    if not draft.content or len(draft.content.strip()) < 200:
        return False
    return True
```

```python
"""FastAPI server that exposes the AI Blog Agent over HTTP."""
import os
import json
import asyncio
from contextlib import asynccontextmanager
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware
from fastapi.responses import JSONResponse
from pydantic import BaseModel

from dotenv import load_dotenv
load_dotenv()

from src.agents.orchestrator import root_blogger
from src.config import get_settings

settings = get_settings()

class TopicRequest(BaseModel):
    topic: str

class BlogResponse(BaseModel):
    status: str
    blog_post: str | None = None
    trace: list[dict] | None = None
    error: str | None = None

@asynccontextmanager
async def lifespan(app: FastAPI):
    # Startup: initialize ADK runner
    app.state.adk_runner = None  # Will be initialized on first request
    yield
    # Shutdown cleanup

app = FastAPI(title="AI Blog Agent", version="0.1.0", lifespan=lifespan)

app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

@app.get("/health")
async def health():
    return {"status": "ok", "model": settings.google_model}

@app.post("/generate", response_model=BlogResponse)
async def generate_blog(request: TopicRequest):
    try:
        # Run the agent (simplified — in production use ADK Runner properly)
        result = await run_agent(request.topic)
        return BlogResponse(status="success", blog_post=result)
    except Exception as e:
        return JSONResponse(
            status_code=500,
            content=BlogResponse(status="error", error=str(e)).model_dump()
        )

async def run_agent(topic: str) -> str:
    """Execute the agent pipeline and return the blog post."""
    # In a real ADK setup, this would use Runner and Session

    from google.adk import Runner
    runner = Runner(

    # This is a simplified representation — actual ADK execution
    # requires async session management
    return f"# Blog post about: {topic}\n\n(Agent pipeline executed successfully)"

@app.get("/")

def start_server():
    import uvicorn
    uvicorn.run(
        "src.server:app",
```

```jsx
import { useState } from 'react'
import './App.css'

function App() {
  const [topic, setTopic] = useState('')
  const [blogPost, setBlogPost] = useState('')
  const [loading, setLoading] = useState(false)
  const [error, setError] = useState('')

  const handleGenerate = async () => {
    if (!topic.trim()) return
    setLoading(true)
    setError('')
    setBlogPost('')

    try {
      const response = await fetch('/generate', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ topic }),
      })
      const data = await response.json()
      if (data.status === 'success') {
        setBlogPost(data.blog_post)
      } else {
        setError(data.error || 'Failed to generate blog post')
      }
    } catch (err) {
      setError('Network error: ' + err.message)
    } finally {
      setLoading(false)
    }
  }

  return (
    <div className="app">
      <header>
        <h1>AI Blog Agent</h1>
        <p>Enter a topic and watch AI agents plan, write, and self-correct</p>
      </header>

      <main>
        <div className="input-section">
          <input
            type="text"
            value={topic}
            onChange={(e) => setTopic(e.target.value)}
            placeholder="e.g., Top 10 use cases for AI agents"
            onKeyDown={(e) => e.key === 'Enter' && handleGenerate()}
          />
          <button onClick={handleGenerate} disabled={loading}>
            {loading ? 'Generating...' : 'Generate Blog'}
          </button>
        </div>

        {error && <div className="error">{error}</div>}

        {blogPost && (
          <div className="output">
            <h2>Generated Blog Post</h2>
            <pre>{blogPost}</pre>
          </div>
        )}
      </main>
    </div>
  )
}

export default App
```

## Risks & Tradeoffs

1. **ADK API changes:** Google ADK is evolving rapidly — pin versions and test after install.
2. **API key management:** Never commit `.env`; use environment variables in production.
3. **Agent execution model:** The actual ADK Runner requires async session management — the server code shows the architecture; production use needs proper session lifecycle.
4. **Frontend simplicity:** This is a functional MVP, not a polished UI. Add streaming, error states, and agent trace visualization for production.
5. **Cost:** Each blog generation triggers multiple LLM calls (planner + checker + writer + checker + orchestrator). Monitor API usage.
